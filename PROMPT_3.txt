Привет! Создай пожалуйста data-слой приложения "Funny Joke App". Данное приложение выводит пользователю шутку, полученную с humor api (https://api.humorapi.com/jokes/random), вместе с которой
с помощью duck api подгружается рандомное изображение утки (можешь использовать https://random-d.uk/api/random или https://random-d.uk/api/randomimg для получения изображения). Humor API для 
получения анекдота требует встроить в query API-KEY. Ключ уже встроен в приложение и может быть получен через BuildConfig.HUMOR_API_KEY. Duck API не требует никакой авторизации. 
Полученную из интернета шутку пользователь может захотеть сохранить в локальное хранилище (RoomDB). Тогда приложение должно сохранить текст шутки и ссылку на асинхронную загрузку изображения утки
в БД. 
Пользователь также может запросить рандомную шутку среди сохранённых в локальном хранилище. Тогда приложение должно вывести пользователю любую из сохранённых шуток и соответствующее ей изображение
утки. Если в БД ещё нет ни одного изображения, то необходимо вывести пользователю предупреждение об отсутствии сохранённых шуток (как Either.fail с ErrorNumber = 6000 и соответствующим описанием ошибки).

Вся работа data-слоя должна вызываться из FunnyJokeRepositoryImpl, где на текущий момент представлена симуляция отправки асинхронных запросов к двум API или к локальному хранилищу (в зависимости от 
функции). 
В рамках выполнения задания ты можешь создавать любые файлы, которые посичтаешь нужным, но соблюдай следующие принципы: всё, что связано с локальным хранилищем должно лежать в data/storage; всё, что связа-
но с сетью - в data/network. Все модели данных (data-классы) должны лежать в models/data. 
Для работы с сетью используй Retrofit и Moshi (если посчитаешь, что требуется десереализация приходящих данных). Для работы с локальным хранилищем - RoomDB. Эти библиотеки уже подключены в dependencies
проекта. На своё усмотрение можешь использовать ещё какие-то другие бибилиотеки/плагины, но тогда их потребуется подключить к проекту самостоятельно.

Также обрати внимание на то, что Duck API выдаёт изображение по протоколу http, а не https, что является неприемлемым для этого проекта. Необходимо написать функцию преобразования ссылки на изображение
утки с протокола http на https при её получении с API. Лучше всего реализовать эту функцию в имплементации репозитория.

Ещё изучи проект на предмет необходимости использования Dependency Injection. Если посчитаешь, что её необходимо использовать в этом проекте, то используй уже подключенный к проекту Dagger Hilt.
Если будешь добавлять DI, то не забудь добавить его в том числе и на UI-слой (MainActivity, FunnyJokeViewModel, FunnyJokeApplication)
